## 1. 產品概述

### 1.1 產品願景

STS (Smart Total Solution) 是一個企業級 AI CEO 智慧決策平台，旨在協助企業主和管理者透過 AI 人工智慧進行數據驅動的商業決策。平台扮演「AI 執行長」的角色，提供即時的商業洞察、KPI 監控、策略分析和決策建議。

### 1.2 核心價值主張

| 價值 | 描述 |
|------|------|
| **智慧決策** | AI 驅動的商業分析與建議，協助快速做出正確決策 |
| **即時洞察** | 即時監控關鍵績效指標，掌握企業營運脈動 |
| **策略框架** | 內建多種商業策略框架，系統化分析企業狀況 |
| **行動導向** | 將分析轉化為具體可執行的行動項目 |

### 1.3 產品定位

- **目標市場**：中小企業、新創公司、企業管理者
- **產品類型**：SaaS 網頁應用程式
- **核心技術**：OpenAI GPT-4o、ChatKit 對話介面
- **設計風格**：現代深色主題、專業商務風格

## 2. 目標用戶

### 2.1 主要用戶角色

#### 角色一：企業創辦人/CEO
- **背景**：中小企業創辦人，需要快速做出商業決策
- **痛點**：缺乏專業顧問團隊、決策資訊不足、時間有限
- **需求**：即時的商業建議、KPI 追蹤、策略規劃支援
- **使用場景**：每日查看 KPI、與 AI 討論策略、追蹤行動項目

#### 角色二：部門主管/經理
- **背景**：負責部門營運的中階主管
- **痛點**：跨部門溝通困難、績效難以量化、向上報告耗時
- **需求**：部門 KPI 監控、自動化報告生成、問題分析
- **使用場景**：週報準備、績效檢討、問題排解

#### 角色三：商業分析師
- **背景**：負責企業數據分析的專業人員
- **痛點**：資料散落各系統、分析工具學習曲線高
- **需求**：整合式數據平台、自然語言查詢、視覺化報表
- **使用場景**：深度分析、趨勢預測、報告製作

### 2.2 用戶需求矩陣

| 需求 | 創辦人/CEO | 部門主管 | 分析師 |

| AI 對話諮詢 | ★★★ | ★★☆ | ★★★ |

| 多語系支援 | ★★☆ | ★★☆ | ★★☆ |

#### 3.2.2 企業診斷模組（Onboarding）

**目的**：收集企業基本資訊，讓 AI 提供客製化的建議

**步驟流程**：

| 步驟 | 名稱 | 內容 |
|:----:|------|------|
| 1 | 歡迎 | 介紹流程概述、預估時間 5-10 分鐘 |
| 2 | 公司檔案 | 公司名稱、產業類別、員工規模、年營收 |
| 3 | 痛點分析 | 主要挑戰、詳細說明、緊急程度 |
| 4 | 數據與目標 | 現有工具系統、數據成熟度、主要目標 |
| 5 | 啟動分析 | AI 分析模擬、進度動畫、準備戰情室 |

**選項配置**：

**產業類別（10 項）**：
1. 科技/軟體
2. 製造業
3. 零售/電商
4. 醫療/健康
5. 金融/保險
6. 專業服務
7. 教育
8. 房地產
9. 餐飲/服務
10. 其他

**員工規模（5 項）**：
1. 新創團隊（1-10 人）
2. 小型企業（11-50 人）
3. 中型企業（51-200 人）
4. 成長型企業（201-500 人）
5. 大型企業（500+ 人）

**年營收（5 項）**：
1. 50 萬以下
2. 50 萬 - 200 萬
3. 200 萬 - 1000 萬
4. 1000 萬 - 5000 萬
5. 5000 萬以上

**主要挑戰（6 項）**：
1. 現金流管理
2. 客戶獲取與留存
3. 營運效率
4. 成長加速
5. 人才與團隊管理
6. 策略規劃

**緊急程度（4 項）**：
1. 緊急 - 需要立即處理
2. 高 - 本季度優先
3. 中 - 今年內解決
4. 低 - 長期規劃

**現有工具系統（8 項，可複選）**：
1. ERP 系統
2. CRM 系統
3. 會計軟體
4. BI 工具
5. 專案管理
6. HR 系統
7. 電商平台
8. 自建系統

**數據成熟度（4 項）**：
1. 進階 - 資料整合完善，有分析團隊
2. 發展中 - 有基本報表，需要更多洞察
3. 基礎 - 資料分散各系統
4. 起步 - 剛開始數位化

### 5.4 OAuth 登入流程

#### Google OAuth
```
用戶點擊「Google 登入」
       ↓
重導向至 Supabase OAuth 端點
       ↓
Google 登入頁面
       ↓
授權成功，回調至應用
       ↓
同步用戶資料至資料庫
       ↓
根據診斷狀態導向
```

#### LINE OAuth
```
用戶點擊「LINE 登入」
       ↓
重導向至 LINE 登入頁面
       ↓
用戶授權
       ↓
回調至 /auth/line/callback
       ↓
後端交換 access token
       ↓
建立/更新用戶資料
       ↓
根據診斷狀態導向
```

```
       ↓

       ↓

       ↓
重新完成企業診斷
```
